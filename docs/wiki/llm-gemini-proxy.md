# Внешняя модель ИИ (Gemini) через прокси — стандарт Shectory

На инфраструктуре Shectory запросы к **Google Gemini API** из прикладных сервисов выполняются по единым правилам: ключ и (при необходимости) **HTTP(S)-прокси** задаются в `.env` на сервере, код вызова — через общий пакет или согласованный клиент.

Цели:

- один набор переменных окружения для всех приложений портала;
- обход региональных/сетевых ограничений через прокси без дублирования логики в каждом репозитории;
- предсказуемое поведение для существующих и **будущих** проектов.

---

## Переменные окружения (прод и разработка)

Имена зафиксированы для всего портала. Подключайте те же ключи в `.env` каждого приложения, которому нужен Gemini.

| Переменная | Обязательность | Назначение |
|------------|----------------|------------|
| `AGENT_LLM_API_KEY` | Да, для вызовов API | Ключ Google AI Studio / Vertex (совместимый с Generative Language API). |
| `AGENT_PROXY` | Для обхода блокировок | URL HTTP(S) прокси, например `http://user:pass@host:port`. Альтернативы: `AGENT_HTTPS_PROXY`, `AGENT_HTTP_PROXY` (любой из трёх читается клиентом). |
| `AGENT_LLM_BASE_URL` | Нет | По умолчанию `https://generativelanguage.googleapis.com/v1beta`. Меняйте только при прокси-шлюзе с другим префиксом. |
| `AGENT_LLM_MODEL` | Нет | Модель по умолчанию для текстовых вызовов, например `gemini-2.5-flash`. |
| `AGENT_LLM_TIMEOUT_MS` | Нет | Таймаут HTTP-запроса к API (мс), по умолчанию 45000. |
| `AGENT_PROXY_CONNECT_TIMEOUT_MS` | Нет | Таймаут установления соединения с прокси (мс), по умолчанию 30000. |

Секреты **не коммитить**; на VDS хранить в `.env` рядом с приложением (как для `DATABASE_URL` и NextAuth).

## Пакет `@shectory/gemini-proxy`

**Исходники и релиз в git:** репозиторий **ourdiary**, каталог:

```text
packages/shectory-gemini/
```

Это общий npm-пакет для экосистемы Shectory (не только дневник). В корне лежит `README.md` со ссылкой на эту вики.

Публичный API (после `npm run build` в каталоге пакета или через `prepare` при установке зависимости):

- `getAgentGeminiEnv()` — разбор `process.env` (тип `AgentGeminiEnv`);
- `isAgentGeminiConfigured()` — есть ли ключ (`AGENT_LLM_API_KEY`);
- `geminiGenerateText(systemInstruction, userText, options?)` — один вызов `generateContent` (системная инструкция + пользовательский текст), опционально переопределение модели и `maxOutputTokens` / `temperature`.

Реализация использует **undici** и `ProxyAgent`: при заданном `AGENT_PROXY` весь исходящий трафик к Gemini идёт через прокси.

### Подключение в Node / Next.js

1. Скопируйте каталог `packages/shectory-gemini` в ваш репозиторий **или** добавьте зависимость через git submodule / приватный npm — важно сохранить **один** исходный код пакета.

2. В `package.json` приложения:

```json
"dependencies": {
  "@shectory/gemini-proxy": "file:./packages/shectory-gemini"
}
```

Путь `file:` укажите относительно корня вашего проекта.

3. Выполните `npm install` (соберётся `dist/` через скрипт `prepare` пакета).

4. Импорт:

```ts
import {
  geminiGenerateText,
  isAgentGeminiConfigured,
} from "@shectory/gemini-proxy";
```

5. На сервере в `.env` задайте `AGENT_LLM_API_KEY` и при необходимости `AGENT_PROXY`.

**Почему пакет лежит в `ourdiary`:** сборка Next.js (Turbopack) корректно подхватывает `file:`-зависимость внутри того же репозитория; вынесение только симлинком за пределы проекта приводит к ошибкам резолва. Другие приложения портала подключают тот же код копированием каталога или общим монорепо.

Новые **простые** текстовые сценарии в Комиссионке предпочтительно подключать через `@shectory/gemini-proxy`, чтобы не дублировать клиент. |
| **Прочие / новые сервисы** | Подключайте пакет и описанные переменные; изменения в общей логике — в `ourdiary/packages/shectory-gemini` и правки этой вики. |

## Эксплуатация и отладка

- При ошибках `fetch` / таймаутах проверьте доступность прокси с хоста приложения и корректность URL.
- Если прокси требует cookie-аутентификацию или нестандартный протокол, обычный HTTP CONNECT-прокси может не подойти — обсуждайте отдельный шлюз или прямой выход (без `AGENT_PROXY`) в доверенной сети.
- Лимиты и квоты Gemini — в консоли Google AI / облака.

## Изменения в пакете

Правки общей логики прокси и env — в репозитории **ourdiary**, каталог `packages/shectory-gemini`; затем обновление зависимости и пересборка во всех потребителях. Эту страницу вики обновляйте при добавлении новых переменных или функций API.
