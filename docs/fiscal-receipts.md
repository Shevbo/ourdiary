# Расходы по QR фискального чека (РФ)

## Контекст

На кассовом чеке РФ QR-код кодирует параметры фискального документа (часто строка вида  
`t=YYYYMMDDTHHmm&s=123.45&fn=...&i=...&fp=...&n=1` или URL с теми же query-параметрами).

**Официальное** открытое API ФНС «проверка чека ККТ» (`kkt-online.nalog.ru`) предполагает регистрацию внешнего пользователя и **мастер-токен** (см. условия на сайте ФНС).

Практичный путь для семейного веб-приложения без бюрократии ФНС — **сторонний сервис проверки чеков** с открытым HTTP API и токеном пользователя (документация **proverkacheka.com**: `POST https://proverkacheka.com/api/v1/check/get`, поля формы `token`, `qrraw` **или** `token`, `qrfile` — файл с фото чека, на котором виден QR). У бесплатного тарифа часто **ограничение по числу запросов** (порядка **15 в сутки**; далее — платный тариф) — лимиты и точные условия на стороне сервиса.

Полная спецификация ответа: в репозитории **`documentation_api.pdf`** — в **`data.json`** позиции в **`items[]`**, суммы позиций и **`totalSum`** указаны **в копейках**; коды ответа `code` (0…5) и поля `user`, `retailPlaceAddress`, `fiscalDriveNumber` и др. — в PDF.

**Токен доступа** выдаётся в личном кабинете сервиса. В `.env` укажите **`PROVERKACHEKA_API_TOKEN`** строкой **без** квадратных скобок, если они есть в инструкции (`[token]` → только `token`). Без токена импорт создаёт **одну** строку расхода по сумме из QR (`s=`).

## Реализация в «Наш дневник»

1. **Клиент**: в форме нового расхода:
   - **QR чека** — модальное окно: **камера** (живой сканер `html5-qrcode`) или **фото из галереи** с распознаванием QR на устройстве (`scanFile`), затем на сервер уходит строка `qrraw`;
   - **Фото чека** — загрузка снимка чека **без** локального сканирования: `multipart/form-data`, поле `file` → сервер отправляет изображение в API как `qrfile` и сохраняет позиции в расходы (нужен токен на сервере).
2. Отдельно для **вложения картинки чека** (не путать с QR): «Снять чек» (камера) и «Чек из галереи» — загрузка в `/api/upload/expense-receipt` с ч/б сжатием.
3. **Сервер** `POST /api/expenses/from-receipt`:
   - JSON `{ "qrraw": "..." }` — как раньше;
   - или **multipart**, поле **`file`** (изображение до ~8 МБ) — только при **`PROVERKACHEKA_API_TOKEN`**;
   - разбор параметров (`src/lib/fns-qr.ts`);
   - если задан `PROVERKACHEKA_API_TOKEN` — запрос к proverkacheka, разбор JSON и извлечение позиций (`src/lib/receipt-proverkacheka.ts`);
   - иначе **fallback**: одна строка расхода на общую сумму из `s=` в QR;
   - автокатегории по названию позиции (`src/lib/receipt-category.ts`);
   - несколько записей `Expense` в одной транзакции + одно начисление рейтинга за импорт.

## Дальнейшее развитие

- Сохранение сырого JSON чека в отдельной таблице для аудита (по согласию пользователя).
