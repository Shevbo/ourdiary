# Наш дневник — краткая инструкция

Версия приложения: **{{VERSION}}**

## Для кого это

Семейное приложение: общая лента событий, календарь, расходы, задачи с сембонами, мечты и режим TV для экрана на стене.

## С чего начать

1. Войдите по **имени для входа** и паролю (или через связку с порталом Shectory — если настроено у администратора).
2. **Лента** — добавляйте события, реагируйте эмодзи, голосуйте, комментируйте.
3. **Расходы** — вносите суммы, категории, при желании прикрепляйте фото или снимок чека (чек сохраняется сжатым в ч/б).
4. **Сембон** — ваши «семейные бонусы» за активность; в истории видно, за что начислили.

## Советы

- На телефоне можно листать вкладки **свайпом** влево/вправо.
- Режим **TV** — для телевизора: крупный текст, без лишних панелей; выход — свайп к ленте или кнопка «дверь» у служебного пользователя.

## Обновления

Раздел **«Что нового в приложении»** в ленте — туда попадают короткие новости после обновлений.

**Автоматически из репозитория:** отредактируйте `src/content/app-news-latest.md` (уберите строку `# SKIP` в начале), напишите текст в духе «что мы тут запилили», затем выполните `npm run sync-app-news` там, где есть рабочий **PostgreSQL** и в **`.env`** или **`.env.local`** указан корректный **`DATABASE_URL`** (тот же, что у приложения). Если в строке подключения остался шаблон `HOST` из примера или БД недоступна с этой машины, скрипт выведет понятную ошибку.

Инструкция для пользователей (`instruction.md`) подставляет версию из `package.json` (сейчас **{{VERSION}}**). После релиза обновите номер в `package.json` — подпись «0.1 MVP» и скачиваемый `.md` обновятся сами.
